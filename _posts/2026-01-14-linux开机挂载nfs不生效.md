```
#/etc/fstab

192.168.0.10:/volume1/appdata  /mnt/appdata  nfs  defaults  0  0
```

开机无法挂载，原因为nfs依赖网络，如果在网络启动前执行，则无法挂载

## 解决方案

### ubuntu

添加 _netdev 使挂载在网络启动之后，只适用于基于systemd的系统

```
#/etc/fstab

192.168.0.10:/volume1/appdata  /mnt/appdata  nfs  defaults,_netdev  0  0
```

### alpine

通过 `localmount` 延迟挂载

```
rc-update add networking boot
rc-update add localmount default
```
